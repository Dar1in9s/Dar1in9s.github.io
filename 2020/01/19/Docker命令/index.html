Docker命令
2020-01-19|docker
字数总计: 1.2k|阅读时长: 4 分钟

Docker 命令有两大类，客户端命令和服务端命令。前者是主要的操作接口，后者
用来启动 Docker daemon。

客户端命令：基本命令格式为 docker [OPTIONS] COMMAND [arg…]
服务端命令：基本命令格式为 docker daemon [OPTIONS]

可以通过 man docker 或 docker help 来查看这些命令。

docker命令

容器生命周期管理

run

创建一个新的容器并运行命令
语法：docker run [options] image [command] [arg...]
常用options说明：

  • -d &emsp;&emsp;后台运行容器，并返回容器id
  • -i &emsp;&emsp;以交互模式运行容器，通常和-t同时使用
  • -t &emsp;&emsp;为容器重新分配一个伪输入终端，通常和-i同时使用
  • -p &emsp;&emsp;指定端口映射，格式为：主机端口:容器端口
  • -P &emsp;&emsp;随机端口映射，容器内部端口随机映射到主机的端口
  • –name=”xxx” &emsp;&emsp;为容器指定一个名称
  • -v,–volume &emsp;&emsp;绑定一个卷
  • –dns 8.8.8.8 &emsp;&emsp;指定容器使用的dns，默认和宿主一致
  • -e xxx=”xxx” &emsp;&emsp;设置环境变量
  • –env-file=[] &emsp;&emsp;从指定文件读入环境变量
  • –expose=[] &emsp;&emsp;开放一个端口或一组端口

start/stop/restart

  • docker start CONTAINER &emsp;&emsp;启动一个或多个已经被停止的容器
  • docker stop CONTAINER &emsp;&emsp; 停止一个运行中的容器
  • docker restart CONTAINER &emsp;&emsp; 重启容器

    rm

删除一个或多个容器
语法：docker rm [options] CONTAINER &emsp;&emsp;
options说明：

  • -f &emsp;&emsp;通过SIGKILL信号强制删除一个运行中的容器
  • -l &emsp;&emsp;移出容器间的网络连接，而非容器本身
  • -v &emsp;&emsp;删除与容器关联的卷

    kill

杀掉一个运行中的容器
语法：docker kill CONTAINER

pause/unpause

docker pause &emsp;&emsp;暂停容器中所有的进程
docker unpause &emsp;&emsp;恢复容器中所有的进程

create

创建一个新的容器但不启动它
语法：docker create

exec

在容器中执行一条命令
语法：docker exec [options] CONTAINER COMMAND [arg...]
options说明：

  • -d &emsp;&emsp;分离模式，在后台运行
  • -i &emsp;&emsp;即使没有附加也保持STDIN打开
  • -t &emsp;&emsp;分配一个伪终端

    容器操作

    ps

列出所有容器
语法：docker ps [options]
options说明：

  • -a &emsp;&emsp;显示所有容器，包括未运行的
  • -f &emsp;&emsp;根据条件过滤显示的内容。
  • –format &emsp;&emsp;指定返回值的模板文件。
  • -l &emsp;&emsp;显示最近创建的容器。
  • -n &emsp;&emsp;列出最近创建的n个容器。
  • –no-trunc &emsp;&emsp;不截断输出。
  • -q &emsp;&emsp;静默模式，只显示容器编号。
  • -s &emsp;&emsp;显示总的文件大小。

    inspect

    docker inspect [options] neme|id &emsp;&emsp;获取容器/镜像的元数据

    top

    docker top &emsp;&emsp;查看容器中运行的进程信息

    attach

    docker attach &emsp;&emsp;连接到正在运行中的容器

    logs

获取容器的日志
语法：docker logs [options] CONTAINER
options说明：

  • -f &emsp;&emsp;跟踪日志输出
  • –since &emsp;&emsp;显示某个开始时间的所有日志
  • -t &emsp;&emsp;显示时间戳
  • –tail &emsp;&emsp;仅列出最新N条容器日志
    docker port CONTAINER &emsp;&emsp;列出指定容器的端口映射

    镜像仓库

    pull

    docker pull NAME[:Tag] &emsp;&emsp;从镜像仓库中拉取或者更新指定镜像

    search

    docker search &emsp;&emsp;从Docker Hub查找镜像

    本地镜像管理

    images

列出本地镜像
语法：docker images [options]
options说明：

  • -a &emsp;&emsp;列出本地所有的镜像
  • –digests &emsp;&emsp;显示镜像的摘要信息

    rmi

删除本地一个或多个镜像
语法：docker rmi [OPTIONS] IMAGE
options说明：

  • -f &emsp;&emsp;强制删除

    build

使用Dockerfie创建镜像
语法：docker build [options] PATH
options说明：

  • –build-arg=[] &emsp;&emsp;设置镜像创建时的变量
  • -f &emsp;&emsp;指定要使用的Dockerfile的路径
  • –pull &emsp;&emsp;尝试去更新镜像的最新版本
  • –rm &emsp;&emsp;设置镜像成功够删除中间容器
  • -t,–tag &emsp;&emsp;镜像的名字及标签，通常name:tag 或者name格式

    history

    docker history IMAGE &emsp;&emsp;查看指定镜像的创建历史

    其他

    info

    docker info &emsp;&emsp;显示 Docker 系统信息，包括镜像和容器数

    version

    docker version &emsp;&emsp;显示 Docker 版本信息。
